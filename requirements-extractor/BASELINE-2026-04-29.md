### spaCy / en_core_web_sm note

The repo-root `requirements.txt` documents both as optional ("only for DDE's NER pass; tests run fine without it"). The 606-test suite confirms that — the rule-based heuristic fallback (10 conservative rules: passive-by-agent, send-to, possessive, compound subject, conditional subject, for-beneficiary, implicit-passive, hyphenated role, between-X-and-Y, role appositive) covers the entire failure surface for this run. This baseline is therefore the **offline-network configuration** (the air-gapped target). A fresh baseline against a spaCy-installed environment would catch any test that diverges between the NLP path and the heuristic-fallback path; we do not have one yet.

## Wording note

The QUEUE.md item title and approved commit message both read "pytest snapshot." The actual harness in `scripts/test_all.sh` is `python3 -m unittest discover tests`, not pytest. This is a known wording inconsistency flagged separately in `PROPOSED.md` (night-auditor 2026-04-29). The harness used for this baseline is the one the suite actually runs under; if the project later adopts pytest, the next baseline should re-capture under that harness because `unittest discover` and `pytest` count tests differently in some edge cases (parameterized tests, doctests, conditional skips).
